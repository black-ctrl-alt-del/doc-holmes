Holmes: Monitoramento Inteligente de Banco de Dados

Holmes é uma poderosa ferramenta de monitoramento de banco de dados projetada para proporcionar a você uma visão completa e precisa do desempenho e integridade do seu sistema de gerenciamento de dados. Com uma série de funcionalidades avançadas, o Holmes se destaca como uma solução abrangente para administradores de banco de dados, oferecendo informações cruciais e insights em tempo real para garantir a estabilidade e eficiência de seu ambiente de dados.

  • Escaneamento e Mapeamento de Tabelas: O Holmes permite uma análise detalhada de todas as tabelas presentes no seu banco de dados. Com um escaneamento abrangente, ele mapeia a estrutura de suas tabelas, identificando relacionamentos e dependências entre elas. Isso proporciona uma visão clara da organização do seu banco de dados.
  • Verificação de Número de Conexões: Mantenha um controle rigoroso sobre o número de conexões ativas ao seu banco de dados. O Holmes alertará sobre picos inesperados de tráfego e ajudará a otimizar o uso dos recursos do sistema.
  • Verificação de Número de Registros: Monitore o crescimento do seu banco de dados acompanhando o número de registros em tempo real. Identifique tendências de crescimento e tome medidas proativas quando necessário.
  • Tamanho do Banco de Dados: O Holmes oferece informações detalhadas sobre o tamanho do seu banco de dados, permitindo que você acompanhe o uso de armazenamento e planeje escalonamentos futuros de acordo com as necessidades do seu sistema.
  • Verificação de Integridade do Serviço: Garanta a disponibilidade contínua do seu banco de dados com verificações automáticas de integridade do serviço. O Holmes identifica possíveis falhas e indisponibilidades, fornecendo alertas em tempo real para que você possa tomar medidas imediatas.
Configuração e Execução

Para a execução da aplicação, será necessário a criação de um arquivo de configuração em formato JSON com o nome config.json no mesmo nível do executável.

Dentro do arquivo, você poderá inserir os serviços a serem monitorados, dentro da estrutura de exemplo:

[ { "database": "", "user":"", "password":"", "host":"", "port":"", "schema": "", "type":"postgresql" }, { "database": "", "user":"", "password":"", "host":"", "port":"", "type":"mysql" } ]

Importante:Para conexões que não exigem a especificação do schema, não definir no objeto de configuração.
